BUFlashController Class Reference

Inherits from NSObject
Declared in BUFlashController.h

Overview

Perform a BlinkUp using screen flashes

The flash controller is used to relay the BlinkUp information to the device using screen flashes.

– presentFlashWithNetworkConfig:configId:animated:resignActive:

Perform a BlinkUp

If using Swift and want enumerated results, please use the Swift extension method

- (void)presentFlashWithNetworkConfig:(nonnull BUNetworkConfig *)networkConfig configId:(nullable BUConfigId *)configId animated:(BOOL)animated resignActive:(nullable FlashResignActiveBlock)resignActive

Parameters

networkConfig

The WifiConfig, WpsConfig, or ClearConfig that is to be performed.
configId

The single use configId for this flashing session. This can be nil in the case of clearing a device
animated

Should the presentation be animated
resignActive

Block that is executed when the BlinkUp screen is dismissed and control is returned to the presenting screen

  preFlashCountdownTime

Seconds before BlinkUp begins

Before the BlinkUp flashes occur there is a countdown to allow the user to prepare the device. The preFlashCountdownTime is the number of seconds that the countdown occurs for. The default is 3 seconds. This property cannot be set lower than 3 seconds, and not more than 10.

@property (nonatomic, assign) NSInteger preFlashCountdownTime

  stringParams

List of strings that can be customized

On occasion it may be useful to change or append to it additional information about the state of the application for the user (such as the device they are about to BlinkUp, or their username). By adding objects into the various stringParams arrays it is possible to inject dynamic content using standard stringWithFormat notation in the localized file string for example “Now connecting device %\@” (remove the backslash if reading source)

Most of these strings can also be modified in the localization file
